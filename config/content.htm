小組倉儲問題

問題發生時間:2021/02/12 上課時

發生問題原因:
本人為bg1之小組組員，平時維護小組網頁時都是先git push 再到倉儲中做Pull requests的動作請組長合併內容，直到今日發生了合併產生衝突。

解決方法:
1.到自己github創一個空的資料夾

2.到小黑窗執行
git clone --recurse-submodules 你們小組的網址

git remote add 資料夾名稱 個人創的github名稱.git

3.到.git資料夾裡的config修改下方資訊

4.在小黑窗中打git push 資料夾名稱

5.到github設定將此save

Heroku建立

1.登入heroku辦帳號，以s+學號，建立第一個app(https://s學號.herokuapp.com)

2.載heroku，放置data裡

3.打開start修改路徑(如圖)

4.在黑窗輸入heroku version確認是否有回應

5.使用小黑窗進到小組倉儲，輸入git ttps://git.heroku.com/s學號.git
6.到config檢查

7.到黑窗輸入git push heroku

8.https://s學號.herokuapp.com確認是否有自己小組倉儲。
